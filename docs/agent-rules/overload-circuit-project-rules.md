# Overload Circuit Project Rules

## When to use
当 Agent 被要求修改《过载回路》代码、文档、规则、数值或提案时，必须先阅读本文件。

## Required Reading Order
1. PROJECT_PRINCIPLES.md
2. docs/current-game-rules.md
3. relevant openspec/specs/*/spec.md
4. relevant openspec/changes/*/proposal.md
5. relevant openspec/changes/*/tasks.md
6. README.md

## Development Protocol
先复述需求；列出会修改的文件；小步修改；不重写架构；不改无关功能；修改后输出测试清单；规则变化必须同步更新 README、规则说明和 docs/current-game-rules.md。

## Forbidden
禁止无授权重写项目、改技术栈、删除现有系统、把核心芯片并入遗物系统、引入后端或在线 API、为了单个功能大改 UI 风格、只改代码不改文档。

## Done Definition
功能可运行；需求已实现；无明显回归；README、规则说明、相关 OpenSpec 文档已更新；提供测试清单。

## Output Format
修改文件列表、修改原因、关键实现说明、测试清单、未完成项。
