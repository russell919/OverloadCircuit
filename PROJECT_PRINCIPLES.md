# 《过载回路》项目原则

## 项目目标
在决赛前完成一个节奏快、反馈强、数值稳定、能展示 AI 协作开发流程的单屏类 Rogue 爆分游戏 Demo。

## 核心体验
抽取模块、热量上升、停手结算、过载风险、达标通关、商店购买遗物、无限挑战。

## 不可违背原则
1. 核心循环必须简单。
2. 每次迭代必须提升试玩体验或开发稳定性。
3. Agent 不允许无授权重写架构。
4. Agent 不允许顺手大改无关功能。
5. 每个新功能必须有明确验收标准。
6. 规则改动必须同步更新 README 和规则说明。
7. 决赛前优先级：稳定性 > 手感反馈 > 数值平衡 > 内容扩展 > 平台移植。
8. 小程序、大量剧情、大量 CG 暂不作为当前主线。
9. AI 辅助开发内容应记录到 docs/ai-collaboration-log.md。
10. 每次修改后必须提供测试清单。

## Agent 工作要求
先阅读本文件，再阅读对应 openspec change；先复述需求和验收标准，再修改代码；只改任务所需文件。
