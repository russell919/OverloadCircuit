# Change: UI 小问题修复

## Why
当前 006-settlement-preview-breakdown 已完成大部分工作，但仍有三个视觉和布局问题需要优化：

1. 公式等号对齐问题："=" 在视觉上没有和数字齐平
2. 滚动条问题：点击"停手结算"时底部会出现滚动条
3. 布局顺序问题："预估结算分数"位置不合理，应该在模块抽取下方

## What
修复三个 UI 小问题：

### 1. 公式等号对齐
- 让"本回合构筑"中的"=" 视觉上与筹码、倍率、X倍率、基础分数字齐平

### 2. 移除停手结算滚动条
- 找到导致宽度变化的组件
- 确保容器宽度稳定，不出现突然的滚动条

### 3. 重新布局顺序
- 调整为：stats-section → breakdown-panel → modules-section → preview-final-section → buttons-section
- "预估结算分数"在模块抽取滚动条下方

## Requirements

### 1. 等号对齐
- 调整 flex 对齐方式或 margin
- "=" 视觉上与数字对齐

### 2. 滚动条移除
- 点击"停手结算"时不出现底部滚动条
- 容器宽度稳定

### 3. 布局调整
- 保持合理的间距
- "预估结算分数"在模块滚动条下方

## Acceptance Criteria
- "本回合构筑"公式中等号视觉上与数字齐平
- 点击"停手结算"时不出现滚动条
- "预估结算分数"、"分数增幅原因"在模块抽取滚动条下方
- 不修改核心玩法和数值
