# Change: 结算预览分解重构

## Why
当前界面中的"预估结算分"仅展示基础公式（筹码 × 倍率 × X倍率），但危险停手奖励、停手触发遗物、结算修正等机制会使实际最终结算分与预估差距很大，甚至相差四五倍。这误导玩家判断，不利于"在爆炸前停手"的核心玩法体验。

## What
重构结算预览 UI 和逻辑，确保预览分数与实际结算分数一致：
1. 将"预估结算分"改名为"当前构筑基础分"，明确其用途
2. 新增"现在停手可得"区域，显示真实预计最终结算分
3. 新增结算分解明细，展示各项影响因素
4. 统一预览和实际结算使用同一套逻辑
5. 调整结算动画围绕新的预估区域展开

## Requirements

### 1. 基础分显示
- 界面中央的分数显示区域改名为"当前构筑基础分"
- 只展示基础公式：筹码 × 倍率 × X倍率
- 不声称是最终结算分

### 2. 停手预览区域
- 新增"现在停手可得"区域
- 显示真正的预计最终结算分
- 必须包含危险停手奖励
- 必须包含停手时触发的遗物影响
- 必须包含所有结算修正

### 3. 结算分解明细
- 明确列出影响最终结算的各项机制
- 每项显示名称、触发条件、影响数值
- 格式清晰，便于玩家理解

### 4. 逻辑一致性
- 预览和实际结算必须使用同一套纯函数
- 不允许 UI 预览和实际结算使用不同逻辑
- 预览分数与最终 roundScore 必须一致

### 5. 结算动画调整
- 结算动画基于新的预估区域展开
- 停手后"现在停手可得"区域进入确认状态或播放动画

## Acceptance Criteria
- 玩家能清晰区分"基础分"和"最终结算分"
- "现在停手可得"数值与实际结算分数一致（误差 < 1）
- 结算分解明细清晰展示各项影响
- 结算动画自然流畅，围绕预估区域展开
- 不修改数值平衡和核心玩法

## Non-goals
- 不修改数值平衡
- 不新增新遗物
- 不修改商店系统
- 不修改核心模块抽取规则
- 不重做整体 UI 风格
