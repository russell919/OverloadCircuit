# 过载回路 Overload Circuit

一款单屏爆分Roguelike游戏，玩家通过抽取模块积累分数，在贪心与稳健之间做出抉择。

## 运行方式

```bash
npm install
npm run dev
```

`npm run dev` 会同时启动 Vite 页面服务和 PVP WebSocket 联机服务，默认监听 `5173` 端口。

### 局域网联机

1. 一台主机运行 `npm run dev`。
2. 主机玩家打开 `http://localhost:5173`，点击 **联机模式**。
3. 同一局域网下的另一台机器访问 `http://主机IP:5173` 或已指向主机 IP 的局域网域名，点击 **联机模式**。
4. 双方进入搜索界面后，后进入的一方通常可以看到先进入玩家广播出的主机，点击该主机即可开始 1v1 对局。

如果使用自定义域名，请确保该域名在局域网 DNS 或 hosts 文件中解析到运行 `npm run dev` 的主机 IP。

## 游戏界面

游戏采用三栏布局：
- **左侧栏**：当前关卡、回合、目标分、累计分、金币、热量条、已拥有遗物
- **中间栏**：筹码/倍率/X倍率数值、预计结算分、模块卡片、抽取/结算按钮
- **右侧栏**：待触发效果、最近抽取、事件日志

## 核心循环

```
主菜单 → 开始游戏
↓
每关循环：
  每回合：抽取模块 / 停手结算
  ↓
  达标即刻通关 / 3回合结束
↓
通关奖励 → 选择遗物 → 商店(每5关) → 下一关
```

### PVP 联机循环

```
主菜单 → 联机模式 → 搜索/加入主机 → 开局全遗物商店
↓
每关 3 回合独立游玩
↓
双方提交本轮成绩并等待
↓
公开双方三回合分数、总分、胜点
↓
随机遗物 → 随机商店 → 下一关
```

## 开局核心芯片系统

PVE 开局核心芯片选择机制已移除，点击“开始游戏”后直接进入关卡。

### 核心芯片列表

| 芯片名称 | 效果 | 适用场景 |
|---------|------|---------|
| 缓冲核心 | 每回合热量上限 +2（原10 → 12） | 提高容错率，适合新手 |
| 增幅核心 | 每回合开始时，初始倍率 = 2（原1） | 提高基础输出 |
| 储能核心 | 每回合开始时，初始筹码 = 40（原0） | 保证最低收益 |
| 裂变核心 | 每回合开始时，初始X倍率 = 2（原1） | 高风险高收益 |

核心芯片不计入遗物数量。

## 关卡规则

- 每关有 **3次回合机会**
- 每回合开始时，筹码/倍率/X倍率重置为初始值
- 基础热量上限为 **10**，核心芯片和遗物可以继续提高上限
- **达标即刻通关**：累计分数达到目标时，立即通关
- 3回合全部结束仍未达标，则游戏结束

### PVP 关卡规则

- PVP 为 1v1 对战，没有每关目标分。
- 每关仍然有 **3 次回合机会**，抽取、停手、过载、遗物触发等机制与 PVE 一致。
- 每名玩家在本地独立完成三回合；未完成前，对手看不到该玩家每回合分数。
- 双方都完成三回合后，同时公开双方总分和三个回合的分数。
- 本轮总分更高的一方获得 1 胜点；若总分相同，双方各获得 1 胜点。
- 任意一方胜点领先对手 3 点时，对局结束，领先者获胜。

### PVP 开局与经济

- PVP 不进行开局核心芯片选择。
- PVP 开局进入全遗物自选商店，初始金币为 **96**。
- 每关结束后仍会出现随机遗物奖励，并且每关都会开启一次随机商店。
- 每关结束金币奖励：16 金币基础奖励 + 本关败方 8 金币补偿 + 当前胜点落后方 `8 × 落后胜点数` 金币补偿。

### 分数计算

```
本回合得分 = 筹码 × 倍率 × X倍率
```

### 热量系统

- 每次抽取模块热量 +1
- 热量达到上限时 **过载爆表**，本回合分数清零
- 部分遗物可在过载时保留部分分数

## 金币系统

通关后获得金币奖励。

### 金币公式

```
金币 = 3 + 剩余回合数 × 2 + 进度奖励
```

- 3：基础通关奖励
- 剩余回合数：未使用的回合数，每回合返还 2 金币
- 进度奖励：每 3 关额外 +1 金币

### 金币获取示例

| 关卡 | 回合 | 基础 | 剩余回合 | 进度 | 合计 |
|-----|------|------|---------|------|------|
| 第1关 | 第1回合通关 | 3 | 4 | 0 | 7 |
| 第1关 | 第2回合通关 | 3 | 2 | 0 | 5 |
| 第1关 | 第3回合通关 | 3 | 0 | 0 | 3 |
| 第4关 | 第1回合通关 | 3 | 4 | 1 | 8 |

## 商店系统

每5关通关后出现商店，可用金币购买遗物。

### 商店规则

- 每次展示 3 个可购买遗物
- 可以购买任意数量买得起的遗物
- 遗物奖励界面选择“跳过（随机获得）”时，会额外获得 4 金币
- 每个商品只能买一次
- 购买的遗物立即加入遗物列表，在后续回合/关卡中按具体效果生效
- 不可重复获得的遗物不会再次出现在商店
- 拥有保险丝后，遗物池中的保险丝会替换为强化保险丝
- 熔断协议、保险丝、量子旁路属于“至多获取一个”的遗物，规则说明界面会显示该特性

### 遗物稀有度分布

商店每个格子独立使用统一稀有度概率：

- 普通：45%
- 稀有：30%
- 史诗：20%
- 传说：5%

## 记录系统

- **最高到达层数**：历史最高到达关卡
- **最高单回合分**：历史单回合最高得分
- **历史最高总分**：历史单局总分
- **排行榜**：服务器运行期间读取本地 `data/leaderboard.json`，显示 PVE 最高单回合分榜和历史最高总分榜前十名
- **玩家档案**：首次进入需填写昵称和四位编号，主菜单可通过“修改昵称”重新编辑

## 操作说明

- **抽取模块**：消耗1热量，随机抽取一个模块
- **停手结算**：将本回合分数加入累计，进入下一回合
- **重新开始**：结束当前游戏，返回主菜单
- 桌面端快捷键：**空格键**抽取模块，**回车键**停手结算

## 与小丑牌的差异

《过载回路》与《小丑牌》都强调构筑、倍率、大数字，但核心玩法有本质区别：

| 维度 | 《小丑牌》 | 《过载回路》 |
|------|-----------|-------------|
| 核心机制 | 出牌组合 | 抽取模块 |
| 风险来源 | 手牌管理 | 热量累积 |
| 策略重心 | 牌型搭配 | 停手时机 |
| 失败条件 | 手牌耗尽 | 过载爆表 |

**本作不是牌型构筑游戏，而是一款"抽取—过热—停手"的超频停手游戏。**

每次抽取模块都会增加热量，让分数变得更强，也更接近爆炸。在爆炸前停手结算，就是你的核心技巧。

### 结算预览系统

游戏提供精确的结算预览，帮助玩家做出明智决策：

- **当前构筑基础分**：显示当前筹码 × 倍率 × X倍率 的基础得分
- **现在停手可得**：显示点击停手后实际会获得的最终分数
- **结算分解明细**：列出所有影响最终得分的遗物和效果

### 危险停手协议
危险停手不再是默认机制。拥有传说遗物 **危险停手协议** 后，在高温时主动停手（热量 >= 70%）可获得额外分数加成：
- 70-80%：+15%
- 80-90%：+30%
- 90%+：+60%

### 超额通关奖励
高分通关可获得额外金币：
- 累计分 >= 目标分 150%：+1 金币
- 累计分 >= 目标分 250%：+2 金币
- 累计分 >= 目标分 400%：+3 金币
